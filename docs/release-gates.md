# 릴리스 게이트(Release Gates)

## 기능(Functional)

- 응시자가 세션을 시작하고, 코드를 편집하고, 제출할 수 있다.
- Platform API가 순서 보장 이벤트 배치를 수락하고 중복을 거부한다.
- Judge가 테스트별로 결정적인(deterministic) 결과를 반환한다.
- Detection이 0이 아닌 모든 신호에 대해 검토자가 읽을 수 있는 근거를 반환한다.
- 관리자/검토자가 원시 근거를 수정하지 않고 세션을 조사할 수 있다.

## 보안(Security)

- Platform API는 제출된 소스 코드를 절대 실행하지 않는다.
- Judge 실행은 기본적으로 네트워크가 차단된 컨테이너 샌드박스를 사용한다.
- CPU, 메모리, 프로세스, 벽시계 제한이 설정되어 있다.
- 비밀값(secret)이 브라우저 번들이나 저장소 파일에 존재하지 않는다.
- 원시 세션 이벤트에 문서화된 보존 정책이 있다.

## 품질(Quality)

- 모든 서비스의 유닛 테스트가 통과한다.
- 계약(contract) 및 종단간(end-to-end) 테스트가 통과한다.
- 프론트엔드 린트 및 프로덕션 빌드가 통과한다.
- 실패 응답이 클라이언트에 스택 트레이스를 노출하지 않는다.
- 오탐(false-positive)이 발생하기 쉬운 정상 워크플로에 대해 Detection 동작을 테스트한다.

## 출시(Launch)

- 최소 100개의 동시 응시자 세션에 대한 부하 테스트를 수행한다.
- 검토자 워크플로와 이의제기(appeal) 정책이 문서화되어 있다.
- 백업 및 복구 절차를 실제로 실행해 본다.
- Detection만으로는 응시자를 자동 탈락시킬 수 없다.
