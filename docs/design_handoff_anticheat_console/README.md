# Handoff: 코딩 테스트 무결성 플랫폼 (Anti-Cheat Assessment Console)

## Overview
부정행위 탐지가 포함된 코딩 테스트 플랫폼입니다. 두 개의 사용자 영역으로 구성됩니다.

- **응시자(Candidate) 플로우** — 입장 → 웹 IDE에서 코딩 → 응시 종료
- **관리자(Admin) 콘솔** — 로그인 → **시험 관리(시험 선택/생성)** → 시험에 입장 후 대시보드 · 시험 설정 · 문제 출제 · 응시자 초대 · 실시간 모니터링 · 근거(위험 평가) 검토

핵심 제품 원칙: **자동 탈락 금지.** 탐지 점수만으로 응시자를 탈락시키지 않으며, 신호별 "근거(evidence)"와 시험 맥락을 사람이 검토합니다.

이번 핸드오프에서 새로 설계된 동작 3가지:
1. 관리자는 로그인하면 먼저 **시험 관리** 페이지로 진입해 새 시험을 만들거나 기존 시험에 입장한다.
2. 문제 출제는 **시험 시작 전에는 편집 가능**, **시험 시작 후에는 잠김(읽기 전용)** 으로 전환된다.
3. 근거(위험 평가) 화면은 사이드바 메뉴에서 진입하지 않고, **모니터링에서 참가자 행을 클릭**하면 해당 응시자의 근거 상세로 이동한다.

## About the Design Files
이 번들의 파일들은 **HTML로 제작된 디자인 레퍼런스**입니다 — 의도한 외형과 동작을 보여주는 프로토타입이며, 그대로 복사해 배포할 프로덕션 코드가 아닙니다. 목표는 이 HTML 디자인을 **대상 코드베이스의 기존 환경(React, Vue, Next.js 등)과 그곳의 확립된 패턴·라이브러리로 재구현**하는 것입니다. 환경이 아직 없다면 프로젝트에 가장 적합한 프레임워크를 선택해 구현하세요.

`Anti-Cheat Console.dc.html`은 "Design Component(.dc.html)" 포맷으로, 모든 스크린이 **라벨이 붙은 프레임**으로 한 캔버스에 나란히 배치되어 있습니다(실제 라우팅된 앱이 아님). `support.js`는 이 포맷의 미리보기 런타임이며 **구현 시 무시**해도 됩니다. 각 프레임은 `data-screen-label` 속성으로 식별됩니다.

## Fidelity
**High-fidelity (hifi).** 최종 색상·타이포그래피·간격·상태가 반영된 시안입니다. 코드베이스의 기존 라이브러리/패턴을 사용하되 아래 디자인 토큰과 측정값을 픽셀에 가깝게 재현하세요. 라이트/다크 테마와 강조색(teal/blue/violet) 토큰이 정의되어 있습니다.

**D4. 문제 — 시작 후·잠김 (관리자-문제-잠김)** — **D3와 동일 레이아웃의 읽기 전용 상태.** 차이점:

  - 좌측 리스트 헤더에 "🔒 잠김" 표시.

**D6. 모니터링 (관리자-모니터링)** — "Live monitor / 참가자 모니터링". **부제(신규)**: "참가자 행을 클릭하면 해당 응시자의 **근거 상세**로 이동합니다." 우상단: "● 10초마다 자동 새로고침" + "고위험만" 필터. 테이블 열 = 응시자(mono) / 상태 / 진행(2/3) / 위험 점수(점수 뱃지[색=위험] + 막대 게이지 max 70px) / "근거 →"(accent). **고위험 행은 --high-soft 배경 강조.** **모든 데이터 행 `cursor:pointer`** — 행 전체가 근거 상세로 가는 진입점. 하단 "위험 점수 높은 순 · 12/12명 표시 중".

**D7. 근거 / 위험 평가 (관리자-근거)** — 모니터링에서 참가자 클릭으로 도달(사이드바 메뉴 아님). 헤더에 사이드바 활성 항목 없음, 대신 **"← 모니터링" 브레드크럼**(accent) + candidate ID(mono). 본문:
  - **위험 평가 요약 카드**: 좌측 "Risk assessment / 위험 평가" + 메타(세션 sess_8f2a1c / 모델 버전 v2.3.1 / 평가 ID ra_77c2, 모두 mono). 우측 "높음" 뱃지(--high) + 대형 점수 **82**/100(42px) + "사람의 검토 권장".
  - **신호별 근거(Evidence ledger)** 리스트: 각 신호 카드 = 번호(mono) + 신호명 + 설명 + 코드(`PASTE_BURST`/`FOCUS_LOSS`) + 점수 뱃지(색=위험). 펼침 영역에 측정값 그리드(붙여넣기 횟수 17 / 최대 길이 1,204자 / 총 시간 3.2s 등, mono) + 임계값 박스(횟수 임계 5, 길이 임계 500).

## Interactions & Behavior
- **로그인 → 시험 관리**: 로그인 성공 시 시험 목록 페이지로 이동(대시보드로 바로 가지 않음).
- **시험 입장**: 카드 "입장 →" → 해당 시험을 활성 컨텍스트로 설정하고 콘솔(대시보드)로 라우팅. 이후 모든 콘솔 화면의 헤더 시험 셀렉트/통계가 이 시험 기준.
- **새 시험 만들기**: 빈 시험 설정 폼으로 이동, 저장 시 목록에 "시작 전" 상태로 추가.
- **출제 잠금 전이**: 시험 상태가 `draft/시작 전` → `진행 중`이 되는 순간(=시험 시작), 문제 화면이 D3(편집) → D4(읽기 전용)로 전환. 시험 종료 전까지 해제 불가. 잠금 상태에서 저장/추가 API 호출은 차단되어야 함(프론트 disabled + 서버 가드 둘 다).
- **모니터링 행 클릭 → 근거**: 행 전체가 클릭 타깃(`cursor:pointer`). 클릭 시 해당 candidate의 근거 화면(D7)으로 이동. "근거 →" 셀도 동일 동작. 초대됨(미응시) 행은 점수 없음("—") — 비활성 처리 권장.
- **자동 새로고침**: 모니터링은 10초 간격 폴링(또는 웹소켓). "고위험만" 토글로 필터.
- **테마 토글(☾)**: 라이트/다크 전환(루트 `data-theme`). 강조색은 teal/blue/violet 선택(`data-accent`).
- 위험 점수 뱃지/게이지 색: 저(<30)=--low, 중(30–69)=--med, 고(≥70)=--high. 임계는 제품 정책에 맞게 조정.

## State Management
- `session.adminUser`, 인증 상태.
- `exams[]`: { id, title, status: 'draft'|'live'|'archived', startAt, endAt, problemCount, participantCount, activeCount }. `status==='live'`일 때 문제 편집 잠금.
- `activeExamId`: 콘솔 컨텍스트.
- `problems[]` per exam: 제목/설명/언어/제한/테스트케이스. 잠금 시 read-only.
- `participants[]` per exam: { candidateId, status, progress, riskScore }.
- `riskAssessment` per session: { score, band, model, signals[] { code, title, score, metrics, thresholds } }.
- 모니터링 실시간 데이터: 폴링/소켓 구독.

## Design Tokens
위 "Global Shell & Design System" 표 참조(색상·라운드·그림자·간격 전부 명시). 핵심 수치: 사이드바 214px, 헤더 60px, 본문 패딩 26/28px, 카드 radius 13–14px, 입력 높이 40–42px, 위험 게이지 높이 5px·max 70px.

## Assets
- 외부 이미지 없음. 모든 아이콘은 **인라인 SVG**(stroke 기반, 24x24 viewBox, stroke-width ~1.6–2). 코드베이스에 아이콘 라이브러리(예: lucide)가 있으면 동일 의미의 아이콘으로 대체 가능: 그리드(대시보드), 슬라이더(시험 설정), 코드 꺾쇠(문제), 사람(응시자), 파형(모니터링), 방패+체크(근거), 자물쇠(잠금), 연필(편집).
- 로고는 텍스트 마크 `SR`(Session Review) / 응시자 측 `AS`. 실제 브랜드 자산이 있으면 교체.
